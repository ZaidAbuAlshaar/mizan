## 1) ترقيات النموذج (Engine 1–2)

| الترقية | الوصف | حالتها عندنا |
|---|---|---|
| U-Net زمني (Spatio-temporal) | يتعلّم توقيع الريّ عبر الموسم بدل عتبة ثابتة | v3 وهدف بعيد · **`review.md` يقترح خطوة وسيطة أرخص:** بصمة الريّ الزمنية anti-phase (GEO-18) فوق v1 |
| التعلّم الذاتي/التبايني (arXiv:2108.05484) | يقلّل التسميات اليدوية ويرفع الدقة حتى 9× | بعيد المدى |

| تصنيف نوع المحصول (WorldCereal + الطيف) | تقدير أدقّ لاستهلاك المياه لكل محصول | بعيد المدى |

## 2) ترقيات التنبّؤ (Engine 3)

| الترقية | الوصف | حالتها عندنا |
|---|---|---|
| LSTM / Temporal Fusion Transformer | متعدّد المتغيّرات (GRACE + أمطار + هكتارات + حرارة) لتنبّؤ أدقّ بنطاق عدم يقين | v3؛ **MVP عندنا: Prophet + backtest + نطاق سنوات** (GEO-10) |
| Downscaling لـ GRACE | ربط TWS الخشن بمؤشّرات سطحية → اقتراب من دقة دون-حوضية | v3؛ **`review.md` ينفّذ بذرته الآن:** طبقة GLDAS `G025/DA1D` + "دفتر الميزان" mass-balance (GEO-11/GEO-15) |
| ربط التنبّؤ بسيناريوهات إدارة | "لو أُغلق X بئر، كيف يتغيّر منحنى الحوض؟" — كما فعل نموذج Springer | بعيد المدى — أقوى ورقة توسّع مؤسسي |

## 3) ترقيات المنتج والبيانات

- **تكامل API مع نظام الوزارة:** سجلّ التراخيص الحقيقي + رفع تقارير التفتيش الميداني → حلقة تغذية راجعة تُحسّن الدرجة.
- **تطبيق ميداني للمفتّش (PWA):** يفتح الموقع المشبوه على الخريطة ويسجّل نتيجة الزيارة من الأرض.
- **كشف شبه آني:** Dynamic World شبه آني + تنبيه فوري عند ظهور حقل جديد بحوض مغلق.
- **وحدة الهبوط الأرضي (InSAR) عبر LiCSAR:** دليل مساند مستقلّ على الاستنزاف (stretch، الدستور v3 §10.6/§18).

## 4) ترقيات المصداقية والتوسّع

- **توسيع بروتوكول التحقّق P7** لكل مواقع الإنفاذ المعلنة تاريخياً → ورقة دقة قابلة للنشر. *(⚠️ `review.md` §2: يجب أولاً حلّ "قنبلة P7" — المواقع الحالية خارج bbox الأزرق — قبل أي توسيع.)*
- **تعميم على أحواض MENA** (نفس المعرّفات، حدود مختلفة) كإثبات قابلية النقل → مسار **AquiferGuard** (`plan.md` §16).
- **بطاقة نموذج (Model Card) + لوحة عدالة:** هل النظام منحاز لمناطق/أحجام معيّنة؟ — تقوّي الثقة المؤسسية (تُعرض في شاشة المنهجية UI-09).
