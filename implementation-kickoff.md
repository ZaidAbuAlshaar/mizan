# ميزان — بروتوكول جولة الديزاين ⇄ التنفيذ

> الجسر بين `design-brief.md` (يُلصق في Claude Design) وتنفيذ الواجهة في Claude Code. اقرأه بعد عودة الديزاين.

## (أ) ما الذي يعود من Claude Design — قائمة تحقّق للمشغّل البشري

ضع كل ما يلي في مجلد `design/approved/` قبل بدء التنفيذ:

- [ ] **الشاشات المعتمدة** (صور/فريمات): الخمس + بانل الدليل، بترتيب أولوية الديمو (١ ← ٣+الدليل ← ٤ ← ٢ ← ٥).
- [ ] **قائمة الـ tokens النهائية:** ألوان (خلفية/أسطح/نص/accent/الخطر-التحذير-السليم) + الخطوط (Almarai/Tajawal) + سُلّم المسافات — جاهزة كـ Tailwind config.
- [ ] **مخزون المكوّنات:** شارات الدرجة 🔴🟠🟢، بطاقة رقم-نطاق، صف الجدول، بطاقة دفتر الميزان بكفّتين، الـ badges («demo data» و«إشارة إقليمية ~٣٠٠كم»)، الأزرار وحالاتها.
- [ ] **أي انحراف عن البريف** موثّق بمبرّره (شاشة/مكوّن + لماذا) — لمراجعة القائد قبل الاعتماد.
- [ ] **فحص القيود الخمسة بصرياً** على الفريمات: صياغة GRACE الإقليمية · شارات demo-data · النطاقات لا العدّادات · «موقع مرشَّح للتفتيش» + زر الحالة · ثنائية اللغة (AR-RTL + EN toggle).

> لا يبدأ التنفيذ قبل أن يعتمد القائد محتوى `design/approved/` وتُغلق كل بنود هذه القائمة.

## (ب) أمر بدء التنفيذ — يُلصق في Claude Code بعد الاعتماد

```
الديزاين معتمد — ملفاته في design/approved/. ابدأ implementation الواجهة الآن:
- نفّذ UI-01→UI-09 من plan.md وفق design-brief.md + الملفات المعتمدة، بترتيب مسار الديمو: شاشة 1 → 3 + بانل الدليل → 4 → 2 → 5.
- Next.js 14 + Tailwind (الـ tokens المعتمدة كـ config) + MapLibre dark + Recharts، RTL أولاً + i18n.
- القيود الحاكمة الخمسة في design-brief.md §2 إلزامية في الكود (صياغة GRACE، demo-data badges، نطاقات لا عدّادات، التموضع، ثنائية اللغة).
- معيار قبول كل شاشة = عمود "معيار الإنجاز" في جدول مهام UI بـ plan.md.
- commit لكل شاشة برسالة ASCII، وحدّث progress.md.
```
